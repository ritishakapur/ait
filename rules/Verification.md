# Verification.md

## Rule: How We Verify
Trust but verify. Every output is checked before it's considered complete.

## The Verification Checklist
Before considering any analysis complete, verify:
1. **Sources** — Every data point has at least one source
2. **Calculations** — Every number is verified (re-calculate)
3. **MECE** — Categories don't overlap and nothing is missing
4. **Consistency** — Numbers and claims don't contradict each other across files
5. **Blind Spots** — Everything uncertain is flagged
6. **Human Checkpoint** — The evaluator questions at the end of each file have been answered

## Cross-File Verification
- Company.md numbers must match Industry.md context
- Stakeholders.md capability framework must match Customers.md needs
- Revenue-Architecture.md buckets must match Revenue-Economics.md math
- Competition.md players must match Industry.md landscape

## The CEO Test
Before finalizing any file, ask: "Would the CEO challenge any claim here?"
- If yes → go back and verify
- If no → still check anyway, because CEOs usually can

## What Good Looks Like
- Zero unverified claims in the final output
- All calculations auditable
- All cross-file references consistent
- All blind spots documented
