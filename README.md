# AI Research & Workflow System

This is a collection of AI agents that help you research companies, build workflows, create templates, and design frameworks. Think of it as a team of specialists working together on your requests.

## How It Works (The Simple Version)

Imagine you ask: "Research Intellect Design Arena."

Here's what happens:

1. **You give a request** — Any kind of research, workflow, template, or framework task.
2. **The Orchestrator (Team Lead)** — Reads your request, figures out what type it is, and picks the right specialist.
3. **The Specialist (Worker)** — Does the actual work. There are different specialists for different jobs:
   - **Research agents** — Study companies, industries, competitors, products, customers, technology, revenue models, and stakeholders.
   - **Workflow agents** — Design step-by-step processes.
   - **Template agents** — Create reusable document templates.
   - **Framework agents** — Build structured thinking models.
4. **The Orchestrator checks the work** — Makes sure it follows the rules (evidence, clear writing, no mistakes).
5. **You review** — The Orchestrator shows you the result and asks if you want to continue, review, or skip.

## The Folder Structure (How Files Are Organized)

Every request gets its own folder. Nothing gets mixed up.

```
Your Project Folder (named after whatever you're working on)
├── Industry.md          (what the Industry agent wrote)
├── Stakeholders.md      (what the Stakeholders agent wrote)
├── Competition.md       (what the Competition agent wrote)
└── ...                  (other files from other agents)
```

**Example:** If you ask to research "Intellect Design Arena", you'll get a folder called `Intellect Design Arena/` with all the research files inside it.

**Important:** The original templates and shared files are NEVER changed. Everything you create goes into your own personal folder.

## The Rules (What Makes This Reliable)

- **Evidence only** — Every claim must be backed by a source. No guessing.
- **Clear writing** — One idea per sentence. No fluff.
- **Trust but verify** — The Orchestrator checks every output before showing it to you.
- **Human in charge** — You confirm at every step. Nothing happens without your say-so.
- **No overwriting** — Shared templates are always protected. Your work stays in your folder.

## How to Use This

### If You're Starting a New Research Project

1. Ask the Orchestrator: "Research [Company Name]" or "Do a full research pipeline on [Company Name]."
2. The Orchestrator will create a folder named after the company.
3. It will send the request to each research agent one by one.
4. After each agent finishes, you'll be asked: "Continue? (yes / review / skip)"
5. Repeat until all agents are done. You'll get a complete research package.

### If You're Starting a Workflow, Template, or Framework

1. Ask the Orchestrator: "Create a [workflow/template/framework] for [subject]."
2. The Orchestrator will create a folder named after the subject.
3. It will send the request to the right specialist agent.
4. You'll review the output and confirm.

### If You're Just Exploring

1. Ask the Orchestrator: "What can you do?" or "Show me the agents."
2. It will list all available agents and what they do.

## The Agents (Who Does What)

### Research Agents (9 total)

| Agent | What It Does |
|---|---|
| **Company** | Profiles the company — financials, hierarchy, assets, market position |
| **Industry** | Studies the market — size, growth, trends, competitors, regulations |
| **Stakeholders** | Maps everyone involved — customers, partners, suppliers, regulators, employees, investors |
| **Revenue-Architecture** | Figures out how the company makes money — revenue streams, pricing, services |
| **Revenue-Economics** | Deep dive into unit economics — costs, margins, market sizing, sensitivity |
| **Competition** | Profiles competitors — strengths, weaknesses, positioning, gaps |
| **Products** | Catalogs products/services — tiers, features, market fit, gaps |
| **Technology** | Maps the tech stack — tools, platforms, integrations, tech gaps |
| **Customers** | Studies customers — segments, needs, journeys, pain points, retention |

### Other Agent Types

| Type | What It Does |

| **Template** | Creates reusable document templates (proposals, reports, etc.) |
| **Framework** | Builds thinking models and structured approaches to problems |

## Key Terms (Plain English)

| Term | What It Means |
|---|---|
| **Orchestrator** | The team lead. Routes requests to the right specialist. Checks work. Asks you for confirmation. |
| **Agent** | A specialist that does a specific type of work. Like a researcher, designer, or planner. |
| **Pipeline** | The order in which agents work. For research, it's Company → Industry → Stakeholders → Revenue → Competition → Products → Technology → Customers. |
| **Parent folder** | The main folder for your project. Named after whatever you're working on. |
| **Subagent** | A specialist that works under the Orchestrator. Each has a specific role. |
| **Template** | A pre-built file format. The original is protected — never overwritten. Your work goes in your own folder. |
| **Evidence** | A source that backs up a claim. Every factual statement needs at least one source. |

## What Happens If Something Goes Wrong

- **Agent fails once** — The Orchestrator retries automatically.
- **Agent fails twice** — The Orchestrator flags it for you to decide what to do.
- **You don't like the output** — You can ask for revisions or skip to the next agent.
- **You want to stop** — Just say "stop" or "skip." Nothing is saved without your confirmation.

## Getting Started

1. Open this workspace in your AI tool.
2. Type a request to the Orchestrator. Examples:
   - "Research Tesla"
   - "Create a workflow for customer onboarding"
   - "Design a framework for market analysis"
   - "Build a template for investor pitch decks"
3. Watch the Orchestrator route your request and delegate to the right agent.
4. Review each output. Confirm, request changes, or skip.
5. Get your complete deliverable in a clean folder.
